About

This site probably does DevOps better than yours.

  • Sourcecontrol

    It all starts with code.

    In this case we are using github, but we can help you integrate other systems with bitbucket or gitlab just as easily. What‽ You aren't using git yet‽ We'll help you migrate.

  • Branches

    You gotta keep it separated.

    If you don't have a plan when it comes to your branching structure, things are going to go wrong. In this case we use simple feature branches that are merged directly into master. We can help you plan a branching structure that is exactly as complex as necessary to meet your requirements.

  • Local Testing

    Don't commit bad code.

    We use docker containers for disposable local test environments that match the production environment and can be created and destroyed in seconds. We have quick running automated tests that can be run by developers locally.

  • Automated Testing

    Don't merge bad code.

    Before pull requests are merged, they are tested automatically and peer reviewed. For this site we use simple testing in cloud.docker.com to test our pull requests. We are fans of drone.io for projects that requires a more complex test system.

  • Build

    Automatic and interactive.

    Builds for your staging environment should be automatic. But sometimes you want to manually trigger your builds. We like to handle interactive builds with git tags. For this site, commits to master are built and deployed to our staging environment. Commits tagged "www" are built and deployed to the production environment. Using tags makes rollbacks a breeze.

  • Deploy

    Put code where it goes.

    When code is built, it should automatically go where it is supposed to. This site uses Docker Cloud to automatically deploy new images to services on an AWS based docker swarm.

  • Notify

    The team knows what is going on.

    We use integrations and custom bots to make sure that our slack channels are updated with pertinent information for anything that goes right or wrong in our DevOps processes.
