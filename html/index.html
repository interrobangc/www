How We Do DevOps

This site probably does DevOps better than yours.

  • Code

    It all starts with code.

    We start by understanding and optimizing code. Great DevOps starts with great code. We are developers who understand systems, not the other way around.

  • Source Control

    Keep track of that great code.

    In this case we are using github, but we can help you integrate our systems with bitbucket or gitlab just as easily. What‽ You aren't using git yet‽ We'll help you migrate.

  • Branches

    You gotta keep it separated.

    If you don't have a plan when it you start branching, things are going to go wrong. For this site we use simple feature branches that are merged directly into master. We can help you plan a branching structure that is exactly as complex as necessary to meet the requirements of your process.

  • Local Testing

    Don't commit bad code.

    We use docker containers to create disposable local environments that duplicate production. They can be created, destroyed, and recreated in seconds. We love quick running lint and smoke tests in pre-commit hooks.

  • Automated Testing

    Don't merge bad code.

    Before pull requests are merged, they are tested automatically and peer reviewed. For this site we use simple testing in cloud.docker.com to test our pull requests. We are fans of drone.io for projects that requires a more complex test system.

  • Build

    Automatic and interactive.

    Some builds need to be automatic. Some builds need to be manually triggered. On this site, commits to master are automatically built and deployed to our staging environment. Commits tagged "www" are built and deployed to the production environment. Using tags makes rollbacks a breeze.

  • Deploy

    Put code where it goes.

    When code is built, it should go where it is supposed to. This site uses Docker Cloud to automatically deploy new images to services on an AWS based docker swarm.

  • Notify

    The team knows what is going on.

    We use integrations and custom bots to make sure that our slack channels are updated with pertinent information for anything that goes right or wrong in our DevOps processes.
